1 Requisitos previos

Asegúrate de tener instalado:

# Node.js 20+
node --version   # debe ser >= 20.0.0

# pnpm 8+
npm install -g pnpm@8
pnpm --version

# PostgreSQL 16 corriendo localmente (o Supabase / Railway)
psql --version

2 Instalar dependencias

cd "d:/CRM BIOGROW/biogrow-platform"
pnpm install

3 Variables de entorno

Crea el archivo apps/web/.env.local con estas variables:

# Base de datos
DATABASE_URL="postgresql://USER:PASSWORD@localhost:5432/biogrow_dev"

# Clerk (obtener en clerk.com → dashboard → API keys)
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY="pk_test_..."
CLERK_SECRET_KEY="sk_test_..."
NEXT_PUBLIC_CLERK_SIGN_IN_URL="/sign-in"
NEXT_PUBLIC_CLERK_SIGN_UP_URL="/sign-up"
NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL="/select-company"
NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL="/select-company"
DATABASE_URL REQUERIDO
Connection string de PostgreSQL. Cambia USER, PASSWORD y nombre de DB.
CLERK_SECRET_KEY REQUERIDO
Clave secreta de Clerk. Solo en servidor. Nunca exponerla al cliente.
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY REQUERIDO
Clave pública de Clerk. Expuesta al cliente (prefijo NEXT_PUBLIC_).

4 Base de datos

Crea la base de datos y aplica el schema:

# Opción A — push directo (desarrollo rápido, sin migración formal)
pnpm db:push

# Opción B — migración formal (recomendado)
pnpm db:migrate

# Generar cliente Prisma (automático en dev, pero útil si cambias schema)
pnpm db:generate

# Poblar datos iniciales (empresas, roles, permisos)
pnpm db:seed
El seed crea las 8 empresas del grupo, sus roles base (ADMIN, MANAGER, REP) y asigna los permisos CRM. Ejecutarlo siempre en un entorno nuevo.

5 Ejecutar en desarrollo

# Desde la raíz del monorepo — levanta TODAS las apps
pnpm dev

# O solo la web app (más rápido)
cd apps/web
pnpm dev

La app queda disponible en:

http://localhost:3000

! Errores comunes

P2021 — Table not found: ejecuta pnpm db:push o pnpm db:migrate.

Clerk 401 — Invalid publishable key: verifica que .env.local existe en apps/web/ (no en la raíz).

Cannot find module '@biogrow/...': ejecuta pnpm install desde la raíz del monorepo.

Port 3000 already in use: usa pnpm dev -- -p 3001 o mata el proceso con npx kill-port 3000.
